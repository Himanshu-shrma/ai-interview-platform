# Evaluation Criteria Per Dimension Per Interview Type

## CODING/DSA Criteria

### problemSolving (weight: 0.20)
**Earns points**: Understanding constraints, identifying edge cases, breaking into sub-problems, asking clarifying questions.
**Loses points**: Misreading the problem, ignoring constraints, jumping straight to code without analysis.

### algorithmChoice (weight: 0.15)
**Earns points**: Selecting correct DS/algorithm, justifying choice with complexity analysis, considering alternatives.
**Loses points**: Brute force without awareness, wrong DS for the problem, can't articulate why.

### codeQuality (weight: 0.15)
**Earns points**: Clean readable code, correct implementation, proper naming, good structure.
**Loses points**: Bugs, poor naming, tangled logic, copy-paste errors.

### communication (weight: 0.15)
**Earns points**: Thinking aloud clearly, explaining decisions, asking good questions, structured approach.
**Loses points**: Long silences without explanation, confusing explanations, not asking questions.

### efficiency (weight: 0.10)
**Earns points**: Stating time/space complexity, optimizing, discussing trade-offs.
**Loses points**: No complexity awareness, inefficient solutions without acknowledgment.

### testing (weight: 0.10)
**Earns points**: Identifying edge cases, manually tracing code, debugging methodically.
**Loses points**: No edge case consideration, "it should work" without verification.

### initiative (weight: 0.10)
**Earns points**: Proactive edge cases, voluntary optimizations, genuine curiosity, asking follow-up questions.
**Loses points**: Only doing minimum requested, no curiosity.

### learningAgility (weight: 0.05)
**Earns points**: Self-correction ("wait, actually..."), adapting after hints, asking "why" questions, generalization.
**Loses points**: Repeating same mistake, not adapting after feedback.

## BEHAVIORAL Criteria (STAR Mapping)

### problemSolving → SITUATION (0.20)
**Earns points**: Clear context setting, specific scenario, relevant example.

### algorithmChoice → TASK (0.15)
**Earns points**: Defined personal role, clear goals, scope of responsibility.

### codeQuality → ACTION (0.15)
**Earns points**: Specific actions taken, detailed steps, impact described.

### communication → RESULT (0.15)
**Earns points**: Quantified outcomes, reflections on learnings, what they'd do differently.

### efficiency → DEPTH (0.10)
**Earns points**: Multiple examples across domains, depth of detail.

### testing → GROWTH (0.10)
**Earns points**: Self-awareness, learning from experience, adaptability.

## SYSTEM_DESIGN Criteria

### problemSolving → Requirements (0.20)
**Earns points**: Functional + non-functional requirements, scope definition, user estimation.

### algorithmChoice → Architecture (0.15)
**Earns points**: Appropriate technology choices, component design, justified decisions.

### codeQuality → Data Design (0.15)
**Earns points**: Data modeling, API design, schema decisions, consistency patterns.

### communication → Trade-offs (0.15)
**Earns points**: Articulating pros/cons, driving the discussion, structured presentation.

### efficiency → Scalability (0.10)
**Earns points**: Bottleneck identification, capacity estimation, horizontal/vertical scaling.

### testing → Reliability (0.10)
**Earns points**: Failure modes, fault tolerance, monitoring, alerting considerations.
